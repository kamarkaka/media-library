# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

A self-hosted video library web app (like a simpler Jellyfin/Plex). Single-owner. Users add directory paths via Settings, scan for video files, browse/search/filter, and stream with resume-playback support. Compatible files (H.264 + AAC/MP3 in mp4/m4v) play directly via HTTP range requests; everything else is transcoded to HLS on-demand with ffmpeg.

## Commands

- `npm run dev` ﻗ°½ start dev server with hot reload (tsx watch), runs on port 3000
- `npm run build` ﻗ°½ compile TypeScript to `dist/`
- `npm start` ﻗ°½ run compiled output (`node dist/index.js`)
- `npm run setup-auth` ﻗ°½ interactive CLI to set login credentials

System dependencies: FFmpeg (`ffprobe` + `ffmpeg`) must be on PATH ﻗ°½ `ffprobe` for video metadata extraction during scan, `ffmpeg` for HLS transcoding during playback. Scraping also needs a Chromium binary: the Puppeteer scrapers use `puppeteer-core` (no bundled browser) and launch via `CHROMIUM_PATH` (default `/usr/bin/chromium-browser`, read directly from `process.env` in `src/scrapers/base/browser.ts`, not surfaced in `src/config.ts`). The Dockerfile installs Chromium and sets `CHROMIUM_PATH`; for local scraping install Chromium and set it yourself.

No test framework is configured.

## Architecture

**Server-rendered app**: Express + EJS templates + vanilla JS on the client. No frontend build step ﻗ°½ Tailwind is loaded via CDN.

**Database**: SQLite via `better-sqlite3` with Knex query builder. Schema is created imperatively in `src/db.ts` (no migrations directory). New columns are added via `ALTER TABLE` checks at startup, not migrations: a hardcoded `[name, type]` list is reconciled against `PRAGMA table_info` for the `videos` table, plus a separate one-off check that adds `alias` to `genres` ﻗ°½ adding a column to any other table needs its own PRAGMA check. Startup also creates indexes: unique `(run_id, video_id, scraper_type)` plus a non-unique `(run_id, scraper_type)` (`idx_coverage_run_scraper`) on `coverage_results`; `(scraper_type, created_at DESC)` on `validation_results`; `videos.release_date/director/maker/label`; `playback_state.last_viewed`; `video_files(video_id)`; and `favorite_moments(video_id)`. Runs in WAL mode with `foreign_keys = ON`. Tables: `videos`, `genres`, `video_genres`, `cast_members`, `video_cast`, `library_paths`, `playback_state`, `settings`, `sessions`, `validation_results`, `coverage_results`, `field_sources`, `scraper_field_config`, `video_files`, `favorite_moments`. A logical entry (`videos` row) can own multiple physical files (`video_files`, one row per file, `video_id` FK ON DELETE CASCADE); `videos.default_file_id` points at the default file and the entry's per-file columns (filename/full_path/codecs/etc.) mirror that default for backward-compatible reads. A boot-time backfill creates one `video_files` row per existing video (idempotent). `favorite_moments` bookmarks one video at one timestamp: `id` (UUID PK), `video_id` (FK videos ON DELETE CASCADE), nullable `file_id` (FK video_files ON DELETE CASCADE), float `timestamp`, `created_at`; the merge worker re-parents losers' moments onto the survivor by updating `video_id`.

**Auth**: Credentials are stored in the `settings` table (not env vars). Login is password-only ﻗ°½ the `auth_username` setting is vestigial: `setup-auth` still prompts for and persists it and `index.ts` loads it into `config.authUsername` (default `'admin'`), but no auth path reads it (`POST /login` in `src/routes/auth.ts` compares only the password's bcrypt hash; `login.ejs` has no username field). On first run, a random session secret and a temporary password are generated; the password (bcrypt-hashed, cost 12) is persisted and printed to console. `npm run setup-auth` (interactive `src/cli.ts`) writes a permanent hash to the DB; `src/cli.ts` also has a `hash-password` subcommand that just prints a hash. Passwords can be changed at runtime via `PUT /api/auth/password` (verifies the current password, re-hashes, updates both the DB and in-memory config ﻗ°½ no restart). Sessions use `express-session` with a custom `SQLiteSessionStore` (`src/session-store.ts`); the cookie is httpOnly, `sameSite: 'lax'`, 30-day (`cookie.secure` is not set, so it travels over plain HTTP), with `resave: false` / `saveUninitialized: false`. The store filters expired rows on read but never prunes them. `requireAuth` (`src/middleware/auth.ts`) returns 401 JSON for `/api/*` and redirects to `/login` otherwise.

**Route structure**: Page routes (`src/routes/*.ts`) render EJS views, mounted in `src/index.ts` with mixed prefixes ﻗ°½ `libraryRouter` and `tagsRouter` at `/` (`/`, `/genres`, `/cast`), `playerRouter` at `/player`, `settingsRouter` at `/settings`, `momentsRouter` (`src/routes/moments.ts`) at `/moments` (server-rendered favorite-moments grid, newest first). API routes are aggregated in `src/routes/api/index.ts` and serve JSON under `/api`. All routes except the public login/logout pages (`src/routes/auth.ts`: GET/POST `/login`, POST `/logout`) require session auth via `requireAuth` (`src/middleware/auth.ts`, mounted after the public auth router). Two gotchas: `/api/library` (`src/routes/api/library.ts`) is the **admin/background-jobs router** (scan, scrape, validate, validate-all, validation-results, coverage, cover-download, merge-dupes, thumbnails, scraper-config, settings, batch-replace, fix-dates, auto-match, db-refresh) ﻗ°½ actual video listing lives in `/api/videos`; and `api/tags.ts` and `api/filters.ts` are both mounted at the `/api` root (not `/api/tags` or `/api/filters`), so they expose top-level paths like `/api/genres`, `/api/cast`, `/api/directors`, `/api/search-suggestions` (tags is mounted first, so its `/genres` and `/cast` GET handlers win). The remaining `/api` sub-routers: `/api/videos` (listing + per-entry edit/delete/stream/hls/thumbnails/frame/file-relink), `/api/playback` (`PUT /:id` upserts position on `video_id` conflict, `GET /recent`, `POST /:id/log`), `/api/paths` (library-path CRUD), `/api/moments` (favorite-moment CRUD + lazily-regenerated `GET /:id/snapshot`), and `/api/auth` (`PUT /password`). Note `auth.ts` exists twice: `src/routes/auth.ts` is the public login/logout page router, `src/routes/api/auth.ts` the authed password-change API.

**Background pipeline**: All long-running work runs in Worker threads spawned by `src/services/scanner.ts`, with progress tracked via module-level singletons polled by the frontend. There are six: `scanProgress`, `scrapeProgress`, `coverageProgress`, `coverDownloadProgress`, `mergeProgress`, `thumbnailProgress`. All reuse the single `ScanProgress` shape (`status: idle | scanning | done | error` ﻗ°½ there is no `'running'` state); each `start*` no-ops if its status is already `'scanning'` (concurrency guard). In dev, `spawnWorker()` constructs an eval Worker that loads `tsx/cjs` first so workers run without a build step.
- **Scan** (`scan-worker.ts`): Walks library paths, discovers video files by extension, inserts new records, runs `ffprobe` for duration/resolution/codec/bitrate/etc. Quick scan only touches new files; full scan re-probes all. Does not fetch external metadata. **Reconcile-then-add ordering**: before adding any new entries, the scan reconciles moved/renamed files ﻗ°½ for every `video_files` row whose `full_path` is no longer on disk (a *stale* row), `relinkMissingFiles` looks among the on-disk files not currently linked to any entry for one whose **code token** (the part of the filename before its first space, lowercased ﻗ°½ the same token scrapers derive) equals the parent entry's `code` (case-insensitive). On a hit it relinks the existing row to that file via the shared `relinkFile` (`merge-helpers.ts`: re-probe + mirror-if-default + drop HLS cache + regenerate thumbnails) and records the path as *consumed*. Consumed paths are then excluded from the new-file pass, so a moved/renamed file fixes its existing entry instead of (a) staying unlinked and (b) being re-added as a duplicate. Matching prefers an exact-basename candidate (a pure move) over a code-only match (a rename), and only entries with a non-empty `code` can be relinked. Both quick and full scans run the reconcile; the `relinked` count is reported on the `ScanProgress` singleton (surfaced in the scan toast). It intentionally **never deletes** records whose files have vanished (`removed` is hardcoded `0`) ﻗ°½ stale `video_files` rows that could *not* be relinked (and their `videos` entries) are kept as "unlinked" so they surface in the Settings missing-files list for relink or deliberate removal; the only deletion is rolling back a brand-new file whose first ffprobe failed.
- **Scrape** (`scrape-worker.ts`): Full scrape re-scrapes all videos; quick scrape only processes records with `matched != 1` (a video becomes `matched = 1` once code, name, cover image, and release date are all present). Supports **per-field scraper config** (`scraper_field_config` table) ﻗ°½ different DB fields can come from different scrapers (fixed set via `FIELD_TO_META`: code, name, release_date, director, maker, label, cover_image, genres, cast). It groups fields by their configured scraper, pre-creates one resolver+scraper instance per needed scraper type (closed in a `finally` after the whole run, not per-video), and for each video invokes every needed scraper, taking each field from its configured scraper's result. Records which scraper sourced each field in `field_sources` and **skips fields marked `manual`** (batch-loaded once to avoid N+1; applies to scalar fields and the genres/cast relation syncs) so user edits are never overwritten. The per-field config is editable from Settings ﻗ│φ Advanced Options and via `GET/PUT /api/library/scraper-config` (empty value deletes the override, falling back to the default scraper). The canonical-ID override is taken only from the default scraper's result. Cover images: with a `code`, `downloadCover` stores a local copy (falls back to the remote URL on failure); without one, the remote URL is stored. The `syncRelation` helper handles many-to-many upserts for genres and cast.
- **Coverage** (`coverage-worker.ts`): Resumable run that reports metadata coverage; results in `coverage_results`. The `run_id` is persisted in the `settings` table under `coverage_run_id` so an incomplete run resumes across restarts (resumability skips `(video_id, scraper_type)` pairs already recorded). Surfaced via `POST /api/library/coverage`, `GET .../coverage/status`, `GET .../coverage/results` (per-scraper success summary for the latest run).
- **Cover download** (`cover-download-worker.ts`): Localizes remote cover URLs ﻗ°½ selects videos whose `cover_image` is still an `http٪` URL and have a non-empty `code`, downloads each via `downloadCover`, and rewrites the column to the local path. It never adds a cover where none exists (videos with no cover, an already-local cover, or no code are skipped).
- **Merge** (`merge-worker.ts` + `merge-helpers.ts`): Collapses entries sharing a non-empty `code` into one ﻗ°½ re-parents the losers' files/genres/cast/field_sources/playback/favorite_moments to a survivor (conflict-safe, survivor's manual edits win), drops the losers' `coverage_results` (no FK; regenerated by coverage runs), recomputes the default file, deletes the losers, and clears each deleted loser's HLS cache via `cleanupCache` ﻗ°½ thumbnails are left alone since they're code-keyed and shared with the survivor. Triggered by the Settings "Merge Duplicate-Code Videos" button and automatically at the end of every scrape.
- **Thumbnail** (`thumbnail-worker.ts` + `thumbnail-generator.ts`): Generates N evenly-spaced JPEG snapshots **per file** via ffmpeg (`-ss` seek, one frame each), N = `thumbnail_count` setting (default 10). Stored **by code** (filesystem is the source of truth, no DB table): `THUMBNAIL_CACHE_DIR/<code>/<code>-<seq>.jpeg`, where `<seq>` is numbered **continuously across the entry's files** (file A ﻗ│φ 001..N, file B ﻗ│φ N+1..2N). Only videos with a non-empty `code` get thumbnails. Exposed per-entry via `POST/GET /api/videos/:id/thumbnails` (regenerates for all the entry's files) and the player's Generate button + carousel.

**Streaming** (`src/services/hls-transcoder.ts`, `src/routes/api/videos.ts`): `player.ts` computes `canDirectPlay` (H.264 + AAC/MP3 + mp4/m4v). Compatible files stream raw via `GET /api/videos/:id/stream` (HTTP range requests). Otherwise the hls.js client requests `GET /api/videos/:id/hls` (master), `/:quality` (variant, triggers ffmpeg transcode), and `/:quality/:segment`. Qualities are 360p/720p/1080p (only those ﻗ┬¤ source height) plus `original`, which is always offered as the highest-bandwidth level regardless of source height (stream-copy when already H.264 + AAC/MP3, else re-encoded with libx264/aac). The variant endpoint rewrites bare `seg\d+.ts` lines in the on-disk `playlist.m3u8` to absolute URLs; `startTranscoding` resolves only once `playlist.m3u8` appears (rejects if ffmpeg exits without it or after a 60s timeout). All streaming endpoints take an optional `?file=<video_files id>` selector; when absent, `resolveFile` (`src/services/video-queries.ts`) picks the entry's default (`default_file_id` ﻗ│φ `is_default` ﻗ│φ legacy videos-row mirror), so an entry's individual files are addressable. `canDirectPlay` and the original stream-copy decision are per file. Gotcha: the player **page**'s server-rendered `canDirectPlay` is computed from `files[0]` (files ordered by filename ASC ﻗ°½ the alphabetically-first file, NOT `default_file_id`), while the client's initial source prefers the `isDefault` file then `files[0]`. Segments cached under `HLS_CACHE_DIR/<videoId>/<fileKey>/<quality>/`; active jobs deduped in a `Map`; `cleanupCache(videoId)` removes a video's cache dir. The client is a **custom player** (`public/js/player.js` + `views/player.ejs`): source-selection fallback chain (directPlay ﻗ│φ hls.js ﻗ│φ native HLS ﻗ│φ raw stream), a **file selector** (when an entry has >1 file), quality selector driven by hls.js levels (Auto/-1), a **thumbnail carousel** (click a snapshot to view it full size in a lightbox; click outside or press Escape to close), custom seek bar, fullscreen (incl. iOS `webkitEnterFullscreen`), keyboard shortcuts, swipe-to-seek (step from the `seek_step` setting), and a click-to-play cover overlay. **Scrub preview**: dragging the seek bar shows a live frame at the dragged position via `GET /api/videos/:id/frame?t=<sec>&file=<id>` ﻗ°½ `src/services/frame-extractor.ts` cuts one JPEG with a fast pre-input `-ss` keyframe seek scaled to 240px, snapping `t` to a 5s grid (`FRAME_STEP`) and caching in a bounded in-memory LRU (`MAX_CACHE=200`, never on disk); the request's AbortSignal (`req.on('close')`) kills a superseded ffmpeg run on disconnect. The client mirrors the 5s rounding (`PREVIEW_FRAME_STEP`, keep in sync), throttles at 150ms, seeks only on release, and the response is `Cache-Control: public, max-age=86400`. **Missing-file / relink**: `player.ts` flags each file `missing` via `fileMissing()` (empty path or not on disk); missing filenames render as red links opening an inline form that `PUT`s `/api/videos/:id/files/:fileId/path`, which validates the new path exists, is a file, lives inside a configured `library_paths` dir, and is unique on `video_files.full_path`, then re-probes + relinks via the shared `relinkFile` (`merge-helpers.ts` ﻗ°½ re-probe + mirror-if-default, drop the HLS cache, and regenerate thumbnails when the entry has a `code`), the same helper the scan worker uses for automatic moved/renamed-file reconcile. **Remove video**: `DELETE /api/videos/:id` deletes the entry (FK cascades video_files/genres/cast/playback/field_sources/favorite_moments; `coverage_results` deleted explicitly), then best-effort clears HLS cache, moment snapshots, and thumbnails+cover (the last only when no other entry shares the `code`); the source media file is never touched. The UI exposes Remove only when an entry has a missing file.

**Favorite Moments** (`src/routes/moments.ts`, `src/routes/api/moments.ts`, `views/moments.ejs`, `public/js/moments.js`): bookmarks of one video at one timestamp (optionally one file), stored in `favorite_moments`. The player's "Save moment" button POSTs `/api/moments` with `{videoId, fileId, timestamp}` (insert + best-effort snapshot ﻗ°½ a snapshot failure does **not** fail the save); moment cards deep-link to `/player/<id>?t=<timestamp>[&file=<fileId>]`, which `player.ts` reads as `startAt`/`startFile` so a non-zero `startAt` overrides the saved resume position and `startFile` selects the initial file. Snapshots live at `MOMENT_CACHE_DIR/<momentId>.jpeg` (keyed by moment id, no DB table; cut via `extractFrame` from `thumbnail-generator.ts`); `GET /api/moments/:id/snapshot` lazily (re)generates a missing one (1-day cache header, 404 if it can't be produced) and the card falls back to the video cover on 404. `snapshotPath(id)` is exported from `api/moments.ts` and reused by `DELETE /api/videos/:id` to delete a removed entry's snapshot jpegs (DB rows cascade via FK, on-disk files don't).

**Scraper plugin system** (`src/scrapers/`): Scrapers are loaded dynamically by directory name (loader `src/scrapers/base/index.ts`); failures fall back to `NoOpScraper`. To add a new scraper, create `src/scrapers/<name>/` with:
- `scraper.ts` ﻗ°½ must export `createScraper(): Scraper` (required). Browser-based scrapers can extend the `PuppeteerScraper` base (`base/puppeteer-base.ts`) and implement `buildUrl()` + `extractMetadata(page)`, and must set the abstract `scraperType` string field (used in `[scraper:<type>]` log prefixes; omitting it fails compilation).
- `config.ts` ﻗ°½ per-scraper constants like `SEARCH_URL_PREFIX` / `SOURCE_URL_PREFIX` (convention used by existing scrapers)
- `resolver.ts` ﻗ°½ must export `resolveSourceUrl(filename: string): Promise<string | null>` (a source-URL string, or `null` when nothing matches ﻗ°½ not a `URL` object); optional `closeResolver(): Promise<void>` (loader defaults it to a no-op when absent)
- `validator.ts` ﻗ°½ must export `getTestConfig(): ValidatorTestConfig` for automated validation (optional)

Existing scrapers: `javtrailers` (default), `123av`, `javxx`, `missav`, `javfilms` ﻗ°½ all five ship a `validator.ts`. The loader special-cases `noop` (and invalid/empty names) ﻗ│φ `NoOpScraper`, and `listScrapers()` auto-discovers available scrapers by scanning the directory (this is how the UI enumerates them, not a hardcoded list). The active scraper is set via `SCRAPER_TYPE` env var. Base types are in `src/scrapers/base/types.ts`. In practice every shipped scraper's `buildUrl()` returns `null` and `PuppeteerScraper.scrape()` uses `sourceUrl || buildUrl(filename)`, so the real page URL comes from `resolver.ts`'s `resolveSourceUrl` ﻗ°½ which searches the site by the filename's leading space-delimited token (`filename.split(' ')[0]`) via `SEARCH_URL_PREFIX`, picks the result whose title `startsWith` that code, and returns its link; `buildUrl` is only a fallback for scrapers that can derive a URL without searching. The resolver owns a module-level `sharedBrowser` separate from the `PuppeteerScraper` instance's own browser ﻗ°½ both launch lazily and are closed independently (`resolver.closeResolver()` and `scraper.close()` are distinct lifecycles).

**Validator cron** (`src/services/validator-scheduler.ts`): On boot (unless `VALIDATOR_ENABLED=false`), schedules `runAllValidations()` on `VALIDATOR_CRON` (default daily 08:00). Each run resolves + scrapes a known test file per scraper, compares against the validator's `expected`, and stores rows in `validation_results` (pruned to a global cap of `30 ﺃ« (scraper count)` most-recent rows by `created_at` ﻗ°½ an aggregate cap, not strictly 30 per scraper).

**Video queries** (`src/services/video-queries.ts`): Centralized query builder for filtering, sorting, pagination, and playback state lookups ﻗ°½ shared by both page routes and API routes. Library browsing uses multi-select **pill filters**: `parseVideoFilters` splits comma-separated query params (`?genre=Action,Drama&cast=Alice,Bob&match=unmatched`) into arrays for genre/director/maker/label/cast plus a `match` filter keyed off the `videos.matched` boolean, and a free-text `q` (the search box) that `applyFilters` matches as one OR group over `videos.filename/code/name` plus EXISTS subqueries on `video_files.filename`, `genres.name`, and `cast_members.name` ﻗ°½ one box searches filename/code/name/per-file-name/genre/cast at once. Sorting has two effective modes only: `last_viewed` (joins `playback_state`, DESC NULLS LAST) and everything else (release_date ASC NULLS LAST). Pagination defaults to `page=1`, `pageSize=24`, but the query param is `page_size` (snake_case); `queryVideos` returns `{ videos, total, page, pageSize, totalPages, hasMore }` with a per-row `file_count` subquery. Also exports `getVideoNeighbors` (prev/next for player navigation) and filter-option lookups for the filter UI.

## Key Patterns

- Config is centralized in `src/config.ts`, sourced from env vars / `.env`. Vars: `PORT` (3000), `DB_PATH` (`data/library.db`), `SCRAPER_TYPE` (`javtrailers`), `VALIDATOR_ENABLED` (true unless `'false'`), `VALIDATOR_CRON` (`0 8 * * *`), `HLS_CACHE_DIR` (`data/hls-cache`), `FFMPEG_PATH` (`ffmpeg`), `COVER_CACHE_DIR` (`data/covers`). Most default data paths are module-relative (`path.join(__dirname, '../data/...')`), so they land under `src/../data` in dev and `dist/../data` after build. `THUMBNAIL_CACHE_DIR` and `MOMENT_CACHE_DIR` are the exceptions ﻗ°½ they default to **absolute** paths (`/data/thumbnail` for thumbnail snapshots and `/data/moments` for favorite-moment snapshots); set both for local dev if `/data` isn't writable. `CHROMIUM_PATH` (scrapers) and `NODE_ENV` (disables Express view cache when not `production`) are read from `process.env` directly, not via `config.ts`. Note: `.env.example` ships `SCRAPER_TYPE=noop`, so copying it gives no-op scraping by default even though the code default is `javtrailers`.
- Auth credentials and the session secret are not env vars ﻗ°½ they live in the `settings` table and are loaded into config at boot. The `settings` table also holds runtime keys editable from the UI without restart: `seek_step` (5/10/15/30), `thumbnail_count` (5/10/15/20/30), `default_scraper`, and `coverage_run_id`.
- Video IDs are UUIDs, but scrapers can override them with a canonical ID
- User edits are never lost: `PUT /api/videos/:id` records each edited field in `field_sources` as `source='manual'` (or the originating scraper when applying scrape results), the scrape worker skips `manual` fields, and the player shows per-field source badges. (`source_url` is a legacy column in `src/db.ts` ﻗ°½ no longer surfaced or written.)
- Playback position is saved per-video and used for resume functionality
- Worker threads prevent concurrent runs via status check on the progress singleton
- Docker support via `Dockerfile` and `docker-compose.yml`: two-stage Node 20 Alpine build (`npm ci`, needs a committed `package-lock.json`), installs ffmpeg + chromium, sets `CHROMIUM_PATH` and `DB_PATH=/data/library.db`, declares a `/data` volume; `docker-compose.yml` mounts `./data:/data` and expects videos at `/media:ro` (edit `/path/to/your/videos`)
